# 코드 실습

### CIFAR-10 CNN(AlexNet 기반) 코드
* 개요: 작은 이미지들(CIFAR-10)을 보고 10개의 종류 중 하나를 맞히는 모델(CNN)를 만드는 과정
* 이미지를 불러오고 → 전처리하고 → CNN 모델을 만들고 → 학습시키고 → 평가하고 → 예측까지 해보는, 딥러닝의 전형적인 전체 흐름을 알수 있다.
*  1. 라이브러리 불러오기
     * 딥러닝 모델을 만들기 위한 도구들을 꺼내는 단계다.
     * TensorFlow/Keras는 딥러닝을 만들 수 있게 해주는 원천기술이고, NumPy는 숫자 계산을, Matplotlib은 그래프를 그리기 위한 도구이다.
     * “집을 짓기 위해 연장통을 여는 것”과 같은 단계라고 보면 된다.
*  2. 데이터 불러오기와 전처리
     * CIFAR-10이라는 작은 그림 데이터셋을 가져온다.
     * 비행기, 자동차, 고양이, 개 등 10종류의 레이블이 붙어 있다.
     * 먼저 그림을 0~255 사이의 값을 0~1 사이의 값으로 바꾼다. 학습이 더 안정적으로 잘 되게 만드는 과정이다.
     * 정답 레이블도 컴퓨터가 이해하기 쉽도록 원-핫 벡터로 바꾼다.즉, “3번 클래스!”라고 숫자로 주는 게 아니라 “10칸 중 딱 하나만 1이고 나머지는 0인 상태”로 바꿔준다.
모델이 여러 클래스 중 하나를 고를 준비를 하는 셈이다.
*  3. 데이터 시각
     * 실제로 이미지 몇 장을 화면에 보여주면서 “데이터가 잘 불러졌나?” “레이블이 맞게 붙어 있나?” 를 눈으로 확인하는 단계다.
     * 딥러닝에서는 이런 기본 점검이 아주 중요하다.
       
*  4. CNN 모델 만들기
     * 이제 본격적으로 “이미지를 보고 특징을 뽑고, 최종적으로 무엇인지 맞히는 뇌”를 만든다.
     * Conv2D 층은 이미지에서 모서리·무늬 같은 기본 특징을 뽑는 역할을 한다.
     * MaxPooling 층은 이미지 크기를 줄이면서 핵심 특징만 남기는 역할을 한다.
     * 이 과정을 여러 번 반복하면 얕은 특징(점·선)부터 깊은 특징(눈·귀·몸 형태)까지 점점 더 복잡한 패턴을 이해할 수 있게 된다.
     * 그 뒤 Flatten을 통해 2차원 이미지 특징을 1차원으로 펼친다.
     * 그리고 Fully Connected Layer(완전 연결층)가 “이 특징들이 모였으니, 이건 고양이 같다” 같은 최종 판단을 내린다.
     * Dropout은 훈련할 때 일부 뉴런을 랜덤으로 끊어서 과적합을 막는 안전장치 역할을 한다.
       
*  5. 모델 컴파일
     * 모델이 “어떤 방식으로 공부할지”를 정한다.
     * Adam이라는 똑똑한 최적화 알고리즘을 사용하고,
     * 정답과 예측의 차이를 계산하기 위해 cross-entropy라는 손실 함수를 쓴다.
     * 모델의 성능은 정확도(accuracy)로 확인한다.
       
*  6. 모델 훈련
     * 실제로 모델에게 데이터를 보여주며 공부시키는 단계다.
     * 전체 데이터를 여러 번 반복해서 보여주는데, 이를 “25 epoch 학습했다”고 표현한다.
     * 학습이 잘 진행되면 손실(loss)은 줄어들고 정확도(accuracy)는 올라간다.
     * 만약 손실이 늘면 학습률이 잘못됐거나 모델 설계에 문제가 있을 수 있다.
       
*   7. 모델 평가
     * 이제 모델에게 한 번도 본 적 없는 테스트 이미지를 보여주고 “실력이 어느 정도인지” 최종 점수를 매긴다.
     * 일반적으로 이 구조에서는 65~75% 정도의 정확도가 나온다.
     * 이 점수는 모델이 새 데이터를 얼마나 잘 맞히는지를 나타낸다.
       
*  8. 학습 과정 그래프 그리기
     * 학습 과정에서 기록된 정보(history)를 사용해 정확도 그래프와 손실 그래프를 그린다.
     * 훈련 정확도와 검증 정확도가 비슷하게 올라가면 정상이다.
     * 훈련만 잘 맞고 검증 정확도는 떨어지기 시작하면 과적합 신호다.
     * 즉, 모델이 “훈련 데이터만 외우고” 새로운 이미지는 못 맞히는 상황이다.
       
*  9. 예측하기
     * 마지막으로 모델에게 실제 이미지를 보여주고 “이건 비행기야!”, “이건 개야!” 같은 예측을 하게 한다.
     * 그림 아래에는 맞히면 파란색, 틀리면 빨간색으로 표시되고 옆에는 각 클래스에 대해 모델이 얼마나 자신 있었는지 확률 막대 그래프도 함께 보여준다.
     * 이 과정을 통해 “모델이 왜 이런 판단을 했는지”도 한눈에 확인할 수 있다.
